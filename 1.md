# QuestGo：現實世界隨機支線任務產生器

本次作業使用 HTML 與 CSS ( Tailwind ) 與 JavaScript 製作一個以地圖定位為主題的網頁應用程式，目的是讓使用者可以接收隨機的「現實支線任務」（如前往某特定顏色遮雨棚的店家閱讀等），為平淡的生活注入新鮮感，解決選擇困難症。

網站主要畫面包含完整的支線任務流程：
- 地圖展示區塊（顯示使用者當前位置）
- 隨機任務提問區
- 載入動畫區塊
- 任務卡片結果區

在功能上，利用 JavaScript 實現了：
- 取得瀏覽器地理定位 (Geolocation API) 功能，並封裝為 Promise
- 整合 Leaflet.js 繪製地圖與顯示「脈衝標記 (Pulsing Marker)」
- 隨機生成各種真實世界的微型任務
- 控制各種動態 UI 狀態切換與動畫（如卡片滑入、Loading 狀態）

網站整體採用具備現代感與擬真 App 體驗的浮動式介面設計。

## 程式碼與說明:

### HTML結構設計:

網站結構主要包含：
- `<div id="map">`：主要地圖顯示區塊
- `<div id="ui-container">`：浮動式介面外層容器
- `<div id="action-container">`：讓使用者點擊產生任務的按鈕區塊
- `<div id="quest-loading">`：顯示任務運算中的過場動畫
- `<div id="quest-card">`：顯示最終隨機任務與詳細內容的結果卡片

使用 Tailwind CSS 類別控制各個 `div` 的顯示（如 `hidden`, `flex` 等）與動畫切換。

### CSS 與 Tailwind 排版:

```html
<div class="absolute bottom-0 w-full p-4 pb-8 transition-transform duration-500 ease-in-out ...">
```

1. 採用 Utility-first 概念，使用 Tailwind CSS 進行排版
2. 製作浮動式任務卡片與漸層按鈕增加設計質感
3. 利用 `transition-transform` 與 `translate-y` 等類別，達成卡片的滑入滑出與流暢動畫效果

### JavaScript 任務與地圖功能:

**模組化的地理定位服務 (`src/services/location.js`):**

```javascript
export const getCurrentLocation = () => {
  return new Promise((resolve, reject) => {
    if (!navigator.geolocation) {
      reject(new Error("此瀏覽器不支援地理定位功能"));
      return;
    }
    navigator.geolocation.getCurrentPosition(
      (position) => resolve({
        lat: position.coords.latitude,
        lng: position.coords.longitude
      }),
      (error) => reject(new Error("無法獲取位置資訊")),
      { enableHighAccuracy: true, timeout: 10000 }
    );
  });
};
```
將原生的 `navigator.geolocation` 封裝為 Promise，避免回呼地獄，並讓主程式更容易地使用 `async/await` 進行非同步處理。

**介面狀態渲染與管理 (`src/ui/render.js`):**

```javascript
export const showLoadingQuest = () => {
  const elements = getUIElements();
  // 隱藏取得任務按鈕，顯示任務卡片
  elements.actionContainer.classList.add('translate-y-[150%]', 'opacity-0');
  elements.questCard.classList.remove('translate-y-full');
  
  // 顯示 Loading 狀態
  elements.questLoading.classList.remove('hidden');
  elements.questLoading.classList.add('flex');
  elements.questContent.classList.add('hidden');
};
```
將 DOM 操作集中至單一模組管理，透過增刪 CSS 類別精準控制卡片的動畫與載入狀態切換。

## AI使用狀況與心得:

### AI使用內容:
在這次專案中，我主要是詢問 Gemini 關於 JavaScript 非同步處理（Promise）的寫法，以及請教 Leaflet 地圖套件在畫面渲染時的一些疑難雜症。

### AI提問與回答摘要:

**提問**
JavaScript 的 navigator.geolocation 抓位置有時候要等很久，我要怎麼讓畫面先顯示載入中的動畫，等確定抓到經緯度之後，才把結果卡片顯示出來？

**AI回答概述**
AI 建議我不要用傳統的 callback，而是用 Promise 把定位功能包起來，並在主程式搭配 async/await 使用。它也教我在執行 await 前，先用 JS 把 loading 區塊的 Tailwind class 改成顯示，等資料回來後再隱藏，這樣就能順利做出等待的過場效果。
